P0 — 必须修复（21 项） 阻断性问题
SEC-01 P0 安全 .env 中硬编码真实 API 密钥，gitignore 未递归覆盖子目录

.env 文件包含真实的 Supabase、Replicate、OpenAI 等密钥，但 gitignore 规则可能无法覆盖 monorepo 子目录中的 .env 文件，存在泄露到 Git 仓库的风险。

影响：密钥泄露可能导致数据库被入侵、AI API 被盗用产生高额费用

SEC-02 P0 安全 文件上传无大小限制和 MIME 类型校验

apps/api/src/routes/uploads.ts

上传接口没有校验文件大小（用户可上传任意大小文件）和 MIME 类型（可上传非图片文件），Zod schema 虽定义但未实际使用。

影响：恶意用户可上传超大文件耗尽存储/带宽，或上传恶意脚本文件

SEC-03 P0 安全 全 API 无速率限制（Rate Limiting）

apps/api/src/app.ts

所有 API 端点没有任何请求频率限制，包括认证端点（/auth/*）、文件上传、AI 服务调用等。任何人可以无限次调用。

影响：暴力破解密码、DDoS 攻击、AI API 费用失控

SEC-04 P0 安全 Services 路由缺少所有权校验（IDOR 漏洞）

apps/api/src/routes/services.ts

Premium 服务接口接受任意 upload_id，未校验该 upload 是否属于当前登录用户。攻击者可以通过遍历 upload_id 查看其他用户的照片和修复结果。

影响：严重的数据泄露，用户隐私照片可被任意访问

SEC-05 P0 安全 错误信息泄露内部实现细节

apps/api/src/app.ts（全局错误处理）

全局错误处理将完整的 Error stack 和内部路径返回给客户端，攻击者可利用这些信息了解服务器架构、文件路径、数据库结构等。

影响：为攻击者提供侦察信息，降低入侵门槛

INF-01 P0 基础设施 无 CI/CD 流水线，所有部署依赖手动 git push

没有 GitHub Actions 或任何自动化 CI/CD。无自动化测试、无代码审查门禁、无构建验证。每次部署完全依赖手动操作。

影响：构建失败可能直接部署到生产环境，回归问题无法提前发现

INF-02 P0 基础设施 无应用错误监控和性能监控（APM）

没有接入 Sentry、Datadog 或任何 APM 工具。生产环境错误只能通过手动查看日志发现，无法实时告警。

影响：用户遇到错误时团队毫不知情，严重影响可靠性感知

INF-03 P0 基础设施 CORS 配置硬编码，无环境变量化

apps/api/src/app.ts

CORS origin 硬编码为 memorylane-web.vercel.app，如需添加测试环境或自定义域名，必须修改代码重新部署。

影响：部署灵活性差，无法支持多环境

INF-04 P0 基础设施 无 HTTPS 强制和 Security Headers

缺少关键安全响应头：Content-Security-Policy、X-Frame-Options、X-Content-Type-Options、Strict-Transport-Security、Referrer-Policy 等。

影响：易受 XSS、点击劫持、MIME 嗅探等攻击

SEO-01 P0 SEO 缺少 Open Graph 和 Twitter Card 标签

apps/web/src/app/layout.tsx

metadata 中完全没有 openGraph 和 twitter 配置。社交平台分享时不会显示预览卡片。

影响：Facebook/Twitter/LinkedIn 分享无图无描述，严重降低社交传播转化率

SEO-02 P0 SEO 缺少结构化数据 JSON-LD

没有任何 JSON-LD Schema 标记（Organization、WebApplication、FAQPage 等），Google 搜索结果无法获得 Rich Snippet。

影响：无法获得星级评分、FAQ 折叠等增强搜索结果展示

SEO-03 P0 SEO 缺少 robots.txt 和 sitemap.xml

项目中不存在 robots.txt、sitemap.xml、app/robots.ts、app/sitemap.ts 等。Next.js 14 原生支持但项目未使用。

影响：搜索引擎爬虫不知道哪些页面可以索引，Dashboard 等私有页面可能被误索引

SEO-04 P0 SEO 子页面缺少独立 metadata

除根 layout 外，所有子页面（/pricing、/login、/services/*、/history 等）均没有独立的 title 和 description。

影响：所有页面 title 重复，Google 认为内容重复度高

SEO-05 P0 SEO 缺少 Canonical URL

全局和各页面均未设置 metadataBase 或 canonical URL，部署域名不确定时可能产生错误的 canonical URL。

影响：潜在重复内容问题

SEO-06 P0 SEO 缺少 favicon、manifest 等 PWA/SEO 基础文件

不存在 favicon.ico、icon.tsx、apple-icon.tsx、manifest.json 等文件。

影响：浏览器标签页无图标，无法添加到主屏幕，OG 分享无图

PERF-01 P0 性能 使用原生 img 标签而非 next/image（结果图）

apps/web/src/app/(dashboard)/restore/[jobId]/page.tsx

修复结果页的 Before/After 图片使用原生 img 标签，无法获得 Next.js 自动优化（懒加载、格式转换、尺寸适配）。

影响：结果图可能很大，加载慢，浪费带宽

PERF-02 P0 性能 无页面级缓存策略

没有任何 revalidate、s-maxage、Cache-Control 配置。静态页面（pricing、landing page）无法利用 CDN 缓存。

影响：每次访问都命中服务器，无法利用 Vercel CDN 边缘缓存

UX-01 P0 UX 缺少全局 404 / not-found 页面

没有 app/not-found.tsx，不存在的路由显示 Next.js 默认 404 页面，与应用设计语言完全不一致。

影响：用户体验断裂，品牌感差

UX-02 P0 UX 缺少全局 error boundary（error.tsx）

没有 app/error.tsx，组件抛出未捕获错误时显示 Next.js 默认错误页面。

影响：运行时错误暴露技术细节，用户体验极差

UX-03 P0 UX 注册后缺少邮箱验证确认页

apps/web/src/app/(auth)/signup/page.tsx

注册成功后直接跳转 /dashboard，没有告知用户需要验证邮箱。如果 Supabase 配置了邮箱验证，用户实际还没完成注册。

影响：用户困惑，注册转化漏斗断裂

UX-04 P0 UX 登录回调失败无错误提示

apps/web/src/app/(auth)/auth/callback/route.ts

Auth callback 失败时重定向到 /login?error=auth_callback_failed，但 login 页面没有读取 error 参数来显示错误信息。

影响：用户登录失败后看到空白登录页，不知道为什么

SEC-06 P1 安全 Zod schema 已定义但未在实际路由中使用

多个路由文件中定义了 Zod 验证 schema，但请求处理中未调用 .parse() 或 .safeParse()。等于验证逻辑形同虚设。

SEC-07 P1 安全 Admin 路由未使用 RBAC 中间件，仅前端检查

Admin API 路由虽有 adminOnlyMiddleware，但部分管理操作（如用户列表）缺少细粒度权限控制。

UX-05 P1 UX Dashboard/History 网络错误只有 console.error，用户无感知

数据加载失败时 catch 块只调 console.error，页面显示为空白，用户不知道发生了什么。

UX-06 P1 UX Sidebar 存在断链（/billing 页面不存在）

apps/web/src/components/layout/sidebar.tsx

"账单" 链接指向 /billing，但项目中不存在该路由页面。

PERF-03 P1 性能 未使用 dynamic import / code splitting

所有组件和库都是静态 import，包括 lucide-react（Dashboard 一次 import 17 个 icon），6 个 Service 页面都有独立 upload 逻辑但没有延迟加载。

INF-05 P1 基础设施 无 API 健康检查端点（/health）

后端没有 /health 或 /readiness 端点，Railway 等平台无法自动检测服务健康状态。

INF-06 P1 基础设施 日志无结构化输出和集中管理

后端日志使用 console.log/error，无结构化 JSON 格式、无日志级别、无集中收集（如 Loki、CloudWatch）。

SEO-07 P1 SEO keywords 标签过于简单，仅 5 个英文关键词

缺少长尾关键词和多语言关键词，应扩充至 15-20 个覆盖核心场景的关键词。

SEO-08 P1 SEO 缺少 hreflang 标签

项目 UI 中英文混用（首页英文，Dashboard 中文），但完全没有 hreflang 标签，Google 无法正确理解多语言关系。

SEO-09 P1 SEO 语义化 HTML heading 层级不完整

多个子页面的 h1 使用 div + text-2xl 样式模拟，缺少语义化标签。CardTitle 渲染为 h3 但用作页面主标题。

SEO-10 P1 SEO Footer 链接指向 # 占位符（About/Blog/Contact/Privacy/Terms）

apps/web/src/components/layout/footer.tsx

所有 Company 和 Legal 链接均为 # 死链，且缺少法律页面可能违反 GDPR/CCPA。
